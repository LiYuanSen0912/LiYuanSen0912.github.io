2. React 事件机制

事件合成

  • React 事件机制分为三个部分：事件注册、事件合成、事件分发
  • 事件注册：React 在组件挂载时，通过原生事件注册，将事件存储在对应的 DOM 节点中
  • 事件合成：React 将原生事件封装成合成事件，将合成事件存储在对应的 DOM 节点中
  • 事件分发：React 在组件更新时，通过事件冒泡，将事件传递给对应的组件
  • 事件合成的好处：减少原生事件的注册，提高性能，统一事件行为

3. React diff 算法

diff 算法

  • React diff 算法是一种高效的 DOM diff 算法，用于比较新旧虚拟 DOM 树，找出需要更新的节点
  • React diff 算法分为三个阶段：树 diff、组件 diff、元素 diff
  • 树 diff：React 将新旧虚拟 DOM 树进行深度优先遍历，找出需要更新的节点
  • 组件 diff：React 将新旧虚拟 DOM 树中的组件进行对比，找出需要更新的组件
  • 元素 diff：React 将新旧虚拟 DOM 树中的元素进行对比，找出需要更新的元素
  • React diff 算法的优点：时间复杂度为 O(n)，性能高，能够快速找出需要更新的节点

4. React Hooks

useState

  • useState 是 React 提供的一个 Hook，用于在函数组件中添加状态
  • useState 接受一个参数作为初始状态，返回一个数组，第一个元素是当前状态，第二个元素是更新状态的函数
  • useState 可以在函数组件中多次调用，每次调用都会创建一个新的状态

useEffect

  • useEffect 是 React 提供的一个 Hook，用于在函数组件中执行副作用操作
  • useEffect 接受两个参数，第一个参数是一个函数，第二个参数是一个数组，用于指定依赖项
  • useEffect 在组件渲染后执行，可以用于执行副作用操作，如数据请求、订阅事件等
  • useEffect 可以在函数组件中多次调用，每次调用都会创建一个新的副作用

useContext

  • useContext 是 React 提供的一个 Hook，用于在函数组件中访问上下文
  • useContext 接受一个上下文对象作为参数，返回上下文对象的当前值
  • useContext 可以在函数组件中多次调用，每次调用都会返回上下文对象的当前值

useMemo

  • useMemo 是 React 提供的一个 Hook，用于在函数组件中缓存计算结果
  • useMemo 接受两个参数，第一个参数是一个函数，第二个参数是一个数组，用于指定依赖项
  • useMemo 在组件渲染后执行，可以用于缓存计算结果，避免不必要的计算

useCallback

  • useCallback 是 React 提供的一个 Hook，用于在函数组件中缓存函数
  • useCallback 接受两个参数，第一个参数是一个函数，第二个参数是一个数组，用于指定依赖项
  • useCallback 在组件渲染后执行，可以用于缓存函数，避免不必要的函数创建

5. React Router

路由模式

  • hash 模式：使用 URL 的 hash 部分（#）来模拟一个完整的 URL，不会向服务器发送请求
  • history 模式：使用 HTML5 的 history API 来实现路由，会向服务器发送请求

路由组件

  • BrowserRouter：使用 HTML5 的 history API 实现路由
  • HashRouter：使用 URL 的 hash 部分（#）来模拟一个完整的 URL
  • Route：定义路由规则，指定路径和对应的组件
  • Link：定义路由链接，点击链接会跳转到对应的路径

路由传参

  • 通过 URL 传参：在路径中使用动态参数，如 /user/:id，然后在组件中使用 useParams() 获取参数
  • 通过 query 传参：在路径中使用 query 参数，如 /user?id=1，然后在组件中使用 useQuery() 获取参数
  • 通过 state 传参：在路径中使用 state 参数，如 /user?id=1，然后在组件中使用 useLocation() 获取参数

路由守卫

  • 通过路由守卫可以实现权限控制、页面跳转等功能
  • 可以使用高阶组件、路由钩子函数等方式实现路由守卫

6. React 性能优化

1. shouldComponentUpdate

  • shouldComponentUpdate 是 React 提供的一个生命周期方法，用于判断组件是否需要更新
  • shouldComponentUpdate 接受两个参数，第一个参数是 nextProps，第二个参数是 nextState
  • shouldComponentUpdate 返回一个布尔值，如果返回 true，则组件会更新，否则不会更新
  • shouldComponentUpdate 可以用于避免不必要的组件更新，提高性能

2. PureComponent

  • PureComponent 是 React 提供的一个组件，用于避免不必要的组件更新
  • PureComponent 继承自 Component，重写了 shouldComponentUpdate 方法，默认比较 props 和 state 是否相等
  • PureComponent 可以用于避免不必要的组件更新，提高性能

3. React.memo

  • React.memo 是 React 提供的一个高阶组件，用于避免不必要的组件更新
  • React.memo 接受一个组件作为参数，返回一个新的组件，新的组件会根据 props 的变化来判断是否需要更新
  • React.memo 可以用于避免不必要的组件更新，提高性能

4. 使用不可变数据

  • 使用不可变数据可以避免不必要的组件更新
  • 可以使用 Immutable.js 库来创建不可变数据

5. 使用懒加载

  • 使用懒加载可以避免一次性加载所有的组件，提高性能
  • 可以使用 React.lazy 和 Suspense 组件来实现懒加载
  • React.lazy 接受一个函数，该函数动态导入组件，返回一个 Promise，Suspense 组件用于等待 Promise 解析，显示加载中的内容

6. 使用服务端渲染

  • 服务端渲染可以提高首屏加载速度，提高用户体验
  • 可以使用 Next.js、Gatsby 等框架来实现服务端渲染

7. 使用 Code Splitting

  • Code Splitting 可以将代码分割成多个小的 chunk，按需加载，提高性能
  • 可以使用 webpack 的 code splitting 功能来实现 Code Splitting

8. 使用 Web Workers

  • Web Workers 可以在后台线程中执行 JavaScript，避免阻塞主线程，提高性能
  • 可以使用 Web Workers 来处理一些耗时的操作，如数据请求、数据处理等
  • 可以使用 workbox 库来实现 Web Workers
  • Web Workers 可以在后台线程中执行 JavaScript，避免阻塞主线程，提高性能
  • 可以使用 Web Workers 来处理一些耗时的操作，如数据请求、数据处理等
  • 可以使用 workbox 库来实现 Web Workers

9. 使用缓存

  • 使用缓存可以避免重复请求，提高性能
  • 可以使用 localStorage、sessionStorage、IndexedDB 等方式来实现缓存
  • 可以使用 workbox 库来实现缓存

10. 使用 CDN

  • 使用 CDN 可以将静态资源部署到多个服务器，提高加载速度
  • 可以使用 CDN 来部署静态资源，如图片、字体、JavaScript、CSS 等

11. 使用压缩

  • 使用压缩可以减少文件大小，提高加载速度
  • 可以使用 webpack 的压缩插件，如 TerserPlugin、UglifyJsPlugin 等
  • 可以使用 gzip、brotli 等方式来实现压缩

12. 使用懒加载图片

  • 使用懒加载图片可以避免一次性加载所有的图片，提高性能
  • 可以使用 IntersectionObserver API 来实现懒加载图片
  • 可以使用 lazyload 库来实现懒加载图片

13. 使用虚拟列表

  • 使用虚拟列表可以避免一次性渲染大量的列表项，提高性能
  • 可以使用 react-virtualized、react-window 等库来实现虚拟列表

14. 使用 WebAssembly

  • WebAssembly 可以将 C/C++、Rust 等语言编写的代码编译成 WebAssembly，提高性能
  • 可以使用 Emscripten 工具将 C/C++、Rust 等语言编写的代码编译成 WebAssembly
  • 可以使用 WebAssembly 来实现一些高性能的计算任务，如图像处理、游戏等
